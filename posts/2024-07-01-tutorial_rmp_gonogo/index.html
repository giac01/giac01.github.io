Introduction

This is a short walk-through of how to calculate relative measurement precision (RMP) internal consistency for d' for a go-nogo task.

For a more detailed tutorial and background on signal detection theory models, I’d recommend the first few chapters of Dection Theory: A User’s Guide and Matti Vourre’s Excellent Blog, which will explain how these models work in more detail.

Load Data from Hedge’s “Reliability Paradox” paper

We will use test-retest data from Craig Hedge’s reliability paradox paper. I will combine data from studies 1 and 2.

The data was downloaded and unzipped from the following page: https://osf.io/cwzds/

Go-nogo Modelling

Condition Response
Go No-Go
Go Hits Misses
No-Go False alarms Correct rejections

We will analyse the go/no-go task using a signal detection theory model to derive a measure performance called sensitivity or d'.

We can derive d’ using a random intercept and slope binomial regression model.

The number of “go” responses will be the outcome variable. Condition (go/nogo) will be our predictor. In the “go” condition, we expect the number of go responses to be higher than the “no-go” condition. The sensitivity or d' parameter measures how well participants shift their probability of responding “go” in the no-go and go conditions.

We can numerically calculate (d’) ((d’)) as a function of hit rate ((HR)) and false alarm rate ((FAR)) is given by:

\[ d' = \Phi^{-1}(HR) - \Phi^{-1}(FAR) \]

Where: ( HR ) is the hit rate (proportion of Go responses in the Go condition) and ( FAR ) is the false alarm rate (proportion of Go responses in the No-Go condition). \(\Phi^{-1}\) represents the z-inverse of the cumulative distribution function of the standard normal distribution. It converts values between 0-1 to z-scores.

We can also model d’ using a random intercept and slope regression probit regression model.

The random intercept captures individual’s propensity to generally respond “go” regardless of condition.

The random slope captures how well individuals’ shift their probability of responding “go” between conditions.

We will contrast code condition so that condition_contrast = -1/2 when “no-go”, and condition_contrast = +1/2 for “go” trials.

This means that the beta coefficient for condition_contrast will correspond the change in the of responding “go” between conditions.

Summarise data

data_long_summarised = data_long %>%
  group_by(pps, session, condition) %>%
  summarise(
    n         = n(),
    n_correct = sum(correct),
    .groups = 'drop'
  ) %>%
  ungroup() %>%
  mutate(
    n_go = ifelse(condition == 0, n_correct, n - n_correct),
    condition_contrast = dplyr::case_when(
      condition == 0 ~ .5,
      condition == 2 ~ -.5
    )
  ) %>%
  ungroup() %>%
  mutate(
      n_go = ifelse(n_go == n, n_go - 1, n_go),
      n_go = ifelse(n_go ==0,                  1  , n_go)
  ) %>%
  arrange(pps)

Instead of modelling each trial individually, we will sum the number of “go” responses in each condition to speed up model fitting.

Because we are assuming that the probability of responding go just varies by condition and participant, we do not need the individual trial data.

For example, if we wanted to estimate out the probability of a bent coin landing on head, we just need the number of heads and number of throws.

We could instead model each individual trial and get the same results (albeit more slowly), particularly if we wanted to include trial-level predictors (e.g., reaction time).

Fit signal detection models on each session (time point)

In the next code chunk, we fit our models to each session (time point) and extract two matrices of our posterior sensitivity draws.

After fitting the models, we extract the posterior draws of each subject’s d’ value at each timepoint into two matrices called subj_draws_time_1 and subj_draws_time_2.

I will stick with brms’s default, vague priors on the model parameters.

For calculating test-retest reliability, I think it is important that we fit the models at each time point independently. Otherwise, data from the second time point will be informing our estimates of d' at the first precise, so they will become more precise (essentially like adding additional trials).

# https://osf.io/cwzds/

model_time_1 = brms::brm(
  n_go | trials(n) ~  1 + condition_contrast + (1 + condition_contrast | pps),
  # family = binomial(link = "probit"),
  family = binomial(link = "logit"),
  data = filter(data_long_summarised, session == 1 ),
  cores   = 2,
  chains  = 2,
  iter    = 10000,
  # init    = .01,
  adapt_delta = .95,
  backend = "cmdstanr",
  threads = threading(4),
  save_warmup = FALSE,
  refresh = 500
)

model_time_2 = brms::brm(
  n_go | trials(n) ~  1 + condition_contrast + (1 + condition_contrast | pps),
  # family = binomial(link = "probit"),
  family = binomial(link = "logit"),
  data = filter(data_long_summarised, session == 2 ),
  cores   = 2,
  chains  = 2,
  iter    = 10000,
  # init    = .01,
  adapt_delta = .95,
  backend = "cmdstanr",
  threads = threading(4),
  save_warmup = FALSE,
  refresh = 500,
)

subj_draws_time_1 = as_draws_df(model_time_1) %>%
                    select(ends_with("condition_contrast]")) %>%
                    t()

subj_draws_time_2 = as_draws_df(model_time_2) %>%
                    select(ends_with("condition_contrast]")) %>%
                    t()


subj_est_time_1 = apply(subj_draws_time_1, 1, mean)

Calculate Relative Measurement Precision

Define the relative_measurement_precision function:

relative_measurement_precision = function(
    input_draws,
    verbose = TRUE
){
  
  input_draws = as.matrix(input_draws)
  
  if (verbose){
    print(paste0("Number of subjects: ", nrow(input_draws)))
    print(paste0("Number of posterior draws: ", ncol(input_draws)))
  }
  
  col_select = sample(1:ncol(input_draws), replace = F)
  input_draws_1 = input_draws[,col_select[1:(length(col_select)/2)]]  
  input_draws_2 = input_draws[,col_select[(length(col_select)/2+1):length(col_select)]] 
  rmp_posterior = sapply(1:ncol(input_draws_1), function(i) cor(input_draws_1[,i],input_draws_2[,i], method = "pearson"))
  hdci = ggdist::mean_hdci(rmp_posterior, .width = 0.95)
  pd = bayestestR::p_direction(rmp_posterior)$pd
  
  return(list(hdci = hdci, pd = pd, rmp_posterior = rmp_posterior,input_draws_1=input_draws_1,input_draws_2=input_draws_2))
}

If there were no changes in individuals latent (true) d' values across the conditions, we would expect the test-rest reliability to correspond to the internal consistency measures. Because change in inevitable, the test-retest reliability should usually be a bit lower than internal consistency.

Because the point-estimates are so similar, the test-retest reliability should be pretty similar too.

This just highlights that there isn’t anything magical about Bayesian or more fancy statistical methods. They don’t by default provide better estimates.

This doesn’t mean that regression approach to calculating d’ is pointless. As we have shown, we can easily calculate internal consistency reliability from our Bayesian regression model. We could also calculate a credible interval for any participant’s true d’ score. We can also handle more complex designs within a regression approach, or we could control for time of day or practice effects, and much more.
